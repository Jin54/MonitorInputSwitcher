# 모니터 입력 전환기

DDC/CI 프로토콜을 사용하여 모니터 입력소스를 쉽게 전환할 수 있는 Windows 프로그램입니다. 여러 모니터를 사용하는 사용자가 DP, HDMI 등 다양한 입력 소스 간 빠른 전환이 필요할 때 유용합니다.

## 프로그램 스크린샷

![모니터 입력 전환기 실행 화면](screenshot.png)

위 스크린샷에서 볼 수 있듯이:
- **상단 그리드**: 각 모니터의 위치와 색상으로 구분된 시각적 표현
- **하단 테이블**: 모니터 정보와 입력소스 선택 드롭다운
- **하단 버튼**: 디스플레이 표시, 새로고침, 종료 기능

실제 사용 화면에서는 Intel(R) UHD Graphics 770 모니터 2대가 감지되어 각각 DP와 HDMI2로 설정된 모습을 확인할 수 있습니다.

## 주요 기능

### 🖥️ 멀티모니터 지원
- **자동 감지**: 연결된 모든 모니터 자동 인식
- **실제 모델명 표시**: "Samsung U28E590D", "LG UltraWide" 등 실제 제품명 표시
- **중복 처리**: 동일한 모델의 모니터 여러 대를 인덱스로 구분

### 🔄 입력소스 관리
- **DDC/CI 프로토콜**: 안정적인 입력 전환을 위한 표준 프로토콜 사용
- **지원 입력**: DP, DP2, HDMI, HDMI2, VGA, DVI
- **현재 상태 표시**: 각 모니터의 현재 입력소스 표시
- **원클릭 전환**: 드롭다운 메뉴로 간편한 입력 변경

### 🎨 시각적 인터페이스
- **그리드 레이아웃**: 모니터 배치를 시각적으로 표현
- **색상 구분**: 각 모니터에 고유 색상 할당으로 쉬운 식별
- **오버레이 표시**: F12 키로 화면에 모니터 이름 표시
- **반응형 UI**: 텍스트 크기 자동 조절 및 여러 줄 표시

### ⌨️ 단축키 지원
- **F12**: 전역 단축키로 앱을 마우스 커서 위치로 이동 (앱이 포커스되지 않아도 작동)
- **클릭으로 닫기**: 오버레이 창을 클릭하면 사라짐

## 기술 세부사항

### DDC/CI 프로토콜
이 프로그램은 DDC/CI(Display Data Channel Command Interface) 프로토콜을 사용하여 모니터와 직접 통신합니다. 이를 통해 소프트웨어에서 입력소스 선택을 포함한 모니터 설정을 제어할 수 있습니다.

### 지원 VCP 코드
- `0x60`: 입력소스 선택
  - `0x0F`: DisplayPort
  - `0x10`: DisplayPort 2
  - `0x11`: HDMI / HDMI 1
  - `0x12`: HDMI 2
  - `0x01`: VGA
  - `0x03`: DVI

## 문제 해결

### 모니터가 감지되지 않음
- 모니터가 DDC/CI를 지원하는지 확인 (모니터 설명서 참고)
- 다른 케이블 타입 시도 (DP가 HDMI보다 일반적으로 잘 작동함)
- 일부 USB-C to DisplayPort 어댑터는 DDC/CI를 지원하지 않을 수 있음

### 입력 전환이 작동하지 않음
- 모니터가 현재 입력에서 DDC/CI를 지원하는지 확인
- 일부 모니터는 설정에서 DDC/CI를 활성화해야 함
- 모니터 목록 새로고침 시도

### 프로그램이 응답하지 않음
- 관리자 권한으로 실행 (일부 시스템에서 필요)
- Windows 이벤트 뷰어에서 자세한 오류 정보 확인
- .NET 8.0 런타임이 설치되어 있는지 확인

## 알려진 제한사항

- **DDC/CI 의존성**: DDC/CI 호환 모니터에서만 작동
- **Windows 전용**: Windows 전용 Win32 API 사용
- **물리적 연결**: 실제 케이블 연결 필요 (무선 디스플레이 지원 안 함)
- **타이밍**: 일부 모니터는 입력 전환에 시간이 오래 걸림
